# web-page-
## Инструкция по сайту «мой сайт 5.0»

Этот проект — **одностраничный сайт (landing)** на HTML/CSS/JS, с переключением языка (RU/EN), калькулятором стоимости и контактной формой.

---

## Быстрый старт

### Как просто открыть сайт локально

1. Откройте файл `мой сайт 5.0.html` двойным кликом (откроется в браузере).
2. Если что-то выглядит «не так», попробуйте открыть через локальный сервер (ниже), а не через `file://`.

### Как запустить через локальный сервер (рекомендуется)

Самый простой вариант без настройки:

- Установите Node.js (если ещё не установлен).
- В терминале в папке проекта выполните:

```bash
npm install
npm run build
```

После сборки откройте `dist/index.html` в браузере.

> Почему так лучше: некоторые вещи в браузере ведут себя стабильнее при открытии через сервер, а не напрямую с диска. Плюс у вас будет «минифицированная» версия файлов в `dist/`.

## Где менять контент (тексты)

На странице многие тексты размечены атрибутом `data-i18n="..."`. Это значит:

- **Тексты лучше менять в `i18n.js`**, чтобы изменения сразу работали и на RU, и на EN.
- В `мой сайт 5.0.html` остаются «базовые» значения, но после загрузки скриптов они подменяются из `i18n.js` в зависимости от выбранного языка.

### Главное место с вашим позиционированием (первый экран)

- Заголовок: ключ `hero.title`
- Подзаголовок: ключ `hero.subtitle`
- Текст: ключ `hero.text`
- Кнопки: `hero.cta1`, `hero.cta2`

И мета‑заголовок вкладки/SEO:

- `meta.title`

## Переводы RU/EN (i18n)

Файл: `i18n.js`

Там два словаря:

- `SITE_I18N.ru` — русские строки
- `SITE_I18N.en` — английские строки

Как добавить новый перевод:

1. В HTML добавьте элементу `data-i18n="my.key"`.
2. В `i18n.js` добавьте `'my.key': 'Текст'` в `ru` и `'my.key': 'Text'` в `en`.

Также поддерживаются:

- **placeholder**: `data-i18n-placeholder="..."` (для `input`/`textarea`)
- **alt**: `data-i18n-alt="..."` (для `img`)

Файлы:

- логика расчёта: `script.js` (функция `calculatePrice`)
- тексты/подписи: `i18n.js` (ключи `calc.*`)

## Контактная форма (отправка)

Файл: `script.js`

Там есть два режима:

1. **Formspree (рекомендуется для хостинга)** — отправка через API Formspree.
2. **Запасной `mailto:`** — если Formspree не настроен или если сайт открыт «с диска».

### Как подключить Formspree

1. Зарегистрируйтесь на Formspree.
2. Создайте форму и получите ID вида `abcxyz...`.
3. В `script.js` замените:

- `const FORMSPREE_FORM_ID = 'YOUR_FORMSPREE_ID';`

на ваш реальный ID.

### Как поменять email для `mailto`

В `script.js` замените:

- `const CONTACT_EMAIL = 'strunka89@yandex.ru';`

Также проверьте email в HTML (в блоке «Контакты»), чтобы всё совпадало.

## SEO и превью для соцсетей

Файл: `мой сайт 5.0.html` (в `<head>`)

Там есть:

- `<meta name="description" ...>`
- `<title>...` (и отдельно ключ `meta.title` в `i18n.js`, который подменяет `document.title`)

Важно перед публикацией:

- заменить `https://example.com/` на ваш реальный домен в OG/Schema/канонических ссылках (если используете).

## Сборка (папка dist)

Команда:

```bash
npm run build
```

Что происходит (см. `build.mjs`):

- минифицируются файлы:
  - `i18n.js` → `dist/i18n.min.js`
  - `script.js` → `dist/script.min.js`
  - `styles.css` → `dist/styles.min.css`
- создаётся `dist/index.html` на основе `мой сайт 5.0.html`, но с заменой ссылок на `.min.*`
- копируются ассеты (если есть):
  - `hero-image.png`, `about-photo.png`

## Частые проблемы

### Форма «не отправляет»

Это нормально, если:

- `FORMSPREE_FORM_ID` не настроен (остался `YOUR_FORMSPREE_ID`)
- сайт открыт как `file://...`

В этих случаях включается запасной вариант через `mailto:` (откроется почтовый клиент).

### Изменили текст, а он «не меняется»

Если у элемента есть `data-i18n`, то после загрузки страницы текст может подменяться из `i18n.js`.

Правильный путь:

- меняйте соответствующий ключ в `i18n.js` в `ru` и (при необходимости) в `en`.
